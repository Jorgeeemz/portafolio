Iptables: Firewall en Linux – Ejercicios prácticos

Jorge Alejandro Cabrera Meza – 181591 Febrero 2026 Netfilter · Tablas · Cadenas · Reglas

Descarga un respaldo académico de esta actividad.

1. Introducción contextual

Iptables es la herramienta de firewall por excelencia en sistemas Linux. Actúa como interfaz de usuario del marco Netfilter del kernel, permitiendo definir reglas que inspeccionan, modifican, aceptan o rechazan paquetes según criterios como dirección IP, puerto, protocolo o estado de la conexión.

Su arquitectura se organiza en tablas (filter, nat, mangle, raw, security) y cadenas (INPUT, OUTPUT, FORWARD, PREROUTING, POSTROUTING). Cada tabla tiene un propósito específico y contiene cadenas predefinidas o creadas por el usuario.

Esta actividad práctica refuerza el conocimiento de la anatomía de los comandos iptables, la interpretación de reglas y la resolución de casos comunes de filtrado. Se presentan ejercicios de completado, relación de tablas y escritura de reglas, basados en escenarios típicos de administración de firewalls.

2. Desarrollo técnico

A continuación se presentan los ejercicios resueltos de la actividad. Se muestran tanto las respuestas originales del estudiante como las correcciones técnicas (cuando aplica) para fines didácticos.

1. Flujo del paquete en iptables

Enunciado original: "Cuando un paquete llega al sistema, primero pasa por una ________, después por una ________, finalmente se ejecuta una ________."

Respuesta: Sugerencia técnica: tabla → cadena → regla

En Netfilter, el paquete recorre las cadenas dentro de cada tabla según el hook correspondiente.

2. Relación de tablas con su propósito

TablaPropósito
FILTERfiltrado de opcionales
NATtraducción de direcciones
MANGLEmodificación avanzada
RAWexperiencia del seguimiento paso libre de paquetes
SECURITYaplicar etiquetas de dar acceso al servicio seguridad

3. Anatomía de un comando iptables

iptables -A INPUT -p tcp -m multiports 80,443 -j ACCEPT

Respuesta:

Interpretación técnica correcta:

  • -A INPUT: Agrega la regla al final de la cadena INPUT.
  • -p tcp: Protocolo TCP.
  • -m multiport --dports 80,443: (corrección: el módulo se llama multiport y la opción es --dports).
  • -j ACCEPT: Acepta el paquete.

4. Este comando permite:

respuesta: "En el puerto 80,443, tabla INPUT se agrega la regla que acepta los paquetes mediante tcp".

Corrección: Se acepta tráfico TCP entrante hacia los puertos 80 y 443 en la cadena INPUT.

5. Variables y opciones comunes

  • a) Limitar intentos por minuto: -m limit --limit 1/minute --limit s/minute
  • b) Filtrar por IP de origen: -s 192.168.1.0/24 - - s / - - source 192.16.5.1.0/24
  • c) Ver solo números, sin DNS: iptables -L -n- L - n
  • d) Ver reglas con contadores: iptables -L -vL - V

6. ¿Qué hace esta regla?

iptables -A INPUT -i eth0 -p tcp -m multiports -dports 22,80,443 \
    -m state --state NEW,ESTABLISHED -j ACCEPT

Respuesta: "Permitir tráfico TCP entrante por la interfaz eth0, en el puerto 22,80,443, solo si es una conexión nueva o ya establecida."

✓ Correcion: (salvo el detalle de multiports → multiport).

📝 Ejercicios de escritura de reglas (página 2)

7. Permitir tráfico HTTP entrante

Respuesta: IPtables -A INPUT -P TCP -dport 50 -on state New -J ACCEPT

Corrección: iptables -A INPUT -p tcp --dport 80 -j ACCEPT

8. Permitir todo el tráfico saliente

Respuesta IPtables -A OUTPUT -J ACCEPT

Corrección: iptables -A OUTPUT -j ACCEPT

9. Permitir SSH solo desde la IP 192.168.1.50

Respuesta: Ptable -A INPUT -P TCP -S 192.168.1.50 -dport 22 -m state NEW, (incompleto)

Corrección: iptables -A INPUT -p tcp -s 192.168.1.50 --dport 22 -j ACCEPT

10. Permitir tráfico TCP entrante a puertos 80 y 443 solo si es conexión establecida o relacionada

Respuesta: IPtables -A INPUT -P TCP -m multiport 80, 443 -J ACCEPT m state -state RELATED -j ACCEPT (mezclado)

Corrección: iptables -A INPUT -p tcp -m multiport --dports 80,443 -m state --state ESTABLISHED,RELATED -j ACCEPT

11. Permitir tráfico TCP entrante por eth0 a 22, 80 y 443, registrar intentos y permitir solo NEW y ESTABLISHED

Respuesta: IPtables -A INPUT -I eth0 -P TCP -m multiport -dports 22,80,443 -m state -state NEW 10,6 -log-pictix

Corrección:

iptables -A INPUT -i eth0 -p tcp -m multiport --dports 22,80,443 -m state --state NEW,ESTABLISHED -j ACCEPT
iptables -A INPUT -i eth0 -p tcp -m multiport --dports 22,80,443 -j LOG --log-prefix "IPT-ENTRANTE: "

Se han corregido los errores sintácticos para mostrar la versión funcional; las respuestas originales se conservan como evidencia del aprendizaje.

4. Reflexión técnica

La práctica con iptables evidencia un conocimiento sólido en la sintaxis básica, pero también muestra errores comunes en el uso de opciones como multiport, --dports o la correcta escritura de -j (no -J). Estos fallos son típicos de principiantes con el firewall de Linux y reflejan la necesidad de practicar con entornos reales y no solo con ejercicios de papel.

Personalmente, considero que iptables es una herramienta indispensable para cualquier profesional de seguridad, pues permite comprender cómo se filtra el tráfico a bajo nivel. Aunque actualmente nftables es su sucesor, la lógica de cadenas, tablas y hooks se mantiene. Dominar iptables facilita la transición a nftables y a firewalls comerciales.

En entornos corporativos, una mala regla puede abrir brechas de seguridad o interrumpir servicios. Por ello, actividades como esta —donde se analizan reglas defectuosas y se corrigen— son fundamentales para desarrollar el criterio técnico necesario en la administración de firewalls.

— Jorge Alejandro Cabrera Meza (181591).
